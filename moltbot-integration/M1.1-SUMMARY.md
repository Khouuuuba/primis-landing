## üîë Key Findings

### 1. No Public Moltbot Docker Image

Moltbot does **not** publish a pre-built Docker image. Deployments must build from source.

**Impact**: We need to either:
- Build and host our own image (GitHub Actions)
- Use platforms that build from Dockerfile (Railway, Fly.io)

**Decision**: Use Railway (builds from Dockerfile directly)

---

### 2. RunPod is Too Expensive for CPU Workloads

RunPod is designed for GPU workloads. CPU-only containers still require renting a GPU.

| Provider | Moltbot Cost/mo | Notes |
|----------|-----------------|-------|
| RunPod (cheapest GPU) | ~$100+ | ‚ùå Overkill |
| Railway | ~$10-15 | ‚úÖ Perfect fit |
| Fly.io | ~$14-20 | ‚úÖ Good alternative |

**Decision**: Use Railway for MVP. Same infrastructure as our backend.

---

### 3. Moltbot Resource Requirements

From official `fly.toml` and Dockerfile:

| Resource | Requirement |
|----------|-------------|
| Node.js | v22+ (strict requirement) |
| Package Manager | pnpm (not npm) |
| Build Tool | Bun (for build scripts) |
| RAM | 2GB minimum |
| CPU | 2 shared vCPUs |
| Storage | Persistent volume for /data |
| Ports | 3000 (gateway), 22 (SSH optional) |

---

### 4. Pricing Model Validated

At **$30/mo** user price with **~$10-15/mo** infrastructure cost:

| Metric | Value |
|--------|-------|
| Infrastructure Cost | ~$12/mo average |
| Price to User | $30/mo |
| Gross Margin | **~60%** |
| Breakeven Users | 1 |

**Conclusion**: Pricing model is viable.
